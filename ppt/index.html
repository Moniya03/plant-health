Introduction & Problem Statement

The Challenge

Traditional plant care relies heavily on reactive, subjective observation. Subtle environmental changes—like microscopic moisture drops or minor temperature shifts—often go entirely unnoticed by human eyes.

By the time physical symptoms (wilting, discoloration) appear on the foliage, the plant has already sustained severe, sometimes irreversible damage.

The Objective

To engineer an automated, proactive monitoring system that captures high-frequency environmental telemetry in real-time.

By integrating Large Language Models (LLMs) with sensor data, the system aims to automatically diagnose plant stress factors and generate actionable care advice before visible damage occurs.

Hardware Components

  • ESP32 DevKit V1: The brain of the IoT edge. Features a dual-core processor and built-in Wi-Fi, allowing seamless continuous data transmission to the local network API.
  • DHT11 Sensor: Provides stable, digital readings for ambient air temperature and relative humidity, critical for assessing the immediate microclimate.
  • HW-103 Moisture Sensor: A resistive analog sensor inserted into the soil. Connected to the ESP32's ADC to precisely track water content percentages.
  • LDR (Photoresistor): Measures ambient light intensity. Used to verify if the plant is receiving adequate illumination for photosynthesis based on its species.

Modern Software Stack

Backend Architecture

  • FastAPI: Provides a high-performance, asynchronous REST API infrastructure for data ingestion.
  • aiosqlite: Ensures non-blocking database I/O operations, preventing bottlenecks during high-frequency sensor updates.

Frontend Interface

  • React 19 & Tailwind: Generates a highly responsive, component-based user interface.
  • Server-Sent Events (SSE): Maintains a unidirectional data stream from server to client for sub-second, real-time dashboard updates without polling.

Key Features & Results

  • Sub-Second Latency: UI updates reflect physical sensor changes almost instantaneously thanks to the optimized SSE connection.
  • Cost-Effective AI Execution: The rule-based gating mechanism successfully filters out normal telemetry, yielding zero LLM API costs during healthy plant conditions.
  • Accurate Stress Diagnosis: The GPT-4o integration correctly identifies simulated scenarios like dehydration and temperature stress with high confidence outputs.
  • Historical Analytics: Seamlessly logs data to SQLite allowing the React dashboard to render detailed trend graphs over time.

Conclusion & Future Scope

The proposed system successfully fuses edge IoT hardware with advanced Generative AI pipelines, shifting plant maintenance from reactive to proactive, automated care.
